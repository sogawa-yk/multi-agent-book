# 章仕様書作成スキル

このスキルは、`docs/book-plan.md`に基づいて各章の仕様書を作成するためのガイドです。

## 前提条件

### 必須ドキュメント
1. `docs/book-plan.md`（書籍企画書）

## 出力先

```
docs/chapter-specs/chNN-spec.md
```

NNは2桁ゼロ埋め（01〜15）

## テンプレートの参照

章仕様書を作成する際は、次のテンプレートを使用してください: ./template.md

## 作成のポイント

### 章間の接続を重視

各章の仕様書には「前の章からの接続」と「次の章への接続」を必ず含める。
読者が章をまたいで学習する際に、文脈の断絶が起きないようにする。

### 図表を先に設計

各節に必要な図表を仕様書の段階で特定する。
これにより執筆時に「図表ファースト」の原則を守りやすくなる。

### 理解度チェック問題の方針

章末に3〜5問の確認問題を設ける。
問題は以下の種類から選択:
- 概念の確認（用語の定義、パターンの特徴）
- 判断問題（ユースケースに対するパターン選択）
- 設計問題（与えられた要件に対するアーキテクチャ設計）
