# タスクリスト

## タスク完全完了の原則

**このファイルの全タスクが完了するまで作業を継続すること**

### 必須ルール
- **全てのタスクを[x]にすること**
- 「時間の都合により別タスクとして実施予定」は禁止
- 未完了タスク（[ ]）を残したまま作業を終了しない

---

## フェーズ1: 過去の申し送り確認

- [ ] 過去の章のtasklist.md振り返りセクションを読み、申し送り事項を抽出
- [ ] 抽出した申し送り事項をrequirements.mdに記載

## フェーズ2: 図表設計

- [ ] {図N.Y}: {図表の内容}
- [ ] {図N.Z}: {図表の内容}

## フェーズ3: 本文執筆

- [ ] {N}.{X} {節タイトル}の執筆
  - [ ] 導入部分
  - [ ] 図表の解説
  - [ ] 具体例
  - [ ] まとめ

- [ ] {N}.{Y} {節タイトル}の執筆
  - [ ] 導入部分
  - [ ] 図表の解説
  - [ ] 具体例
  - [ ] まとめ

## フェーズ4: 章の導入と結び

- [ ] 章の導入（前章との接続）
- [ ] 章の結び（次章への橋渡し）

## フェーズ5: 理解度チェック問題

- [ ] 問題1: {概要}
- [ ] 問題2: {概要}
- [ ] 問題3: {概要}

## フェーズ6: セルフレビュー

- [ ] 図表の完全性チェック（各節に最低1つ）
- [ ] 用語統一チェック（glossary.md照合）
- [ ] 文体チェック（常体統一、文の長さ）
- [ ] 図表番号・参照の整合性チェック
- [ ] 過去の申し送り事項が反映されているか確認
