# 章のレビュー

引数: 章番号(例: `/review-chapter 4`)

## 実行方法

```bash
claude
> /review-chapter 4
```

## 手順

### ステップ1: 原稿の存在確認

`manuscript/ch[章番号]/ch[章番号].md` が存在するか確認します。

### ステップ2: chapter-reviewerサブエージェント起動

Task toolを使用してchapter-reviewerサブエージェントを起動してください:
- subagent_type: "chapter-reviewer"
- description: "Chapter detailed review"
- prompt: 以下のドキュメントを読み込んだ上で、`manuscript/ch[章番号]/ch[章番号].md` を詳細にレビューしてください。

  参照ドキュメント:
  - `docs/chapter-specs/ch[章番号]-spec.md`（章の仕様書）
  - `docs/writing-guidelines.md`（執筆ガイドライン）
  - `docs/glossary.md`（用語集）
  - 前後の章の原稿

  以下の観点で評価してください：

  **1. 仕様との整合性**
  - chapter-specに定義された節がすべて含まれているか
  - 章の目的が達成されているか
  - 図表リストの図表がすべて含まれているか
  - 目標ページ数との乖離

  **2. 用語の統一**
  - glossary.mdに定義された用語が正しく使われているか
  - 初出時の表記ルール（日本語（英語）形式）が守られているか
  - 未定義の専門用語がないか

  **3. 文体・表記**
  - 常体（である調）で統一されているか
  - 1文の長さ（60字以内を推奨）
  - 段落の長さ（5文以内を推奨）
  - 接続詞の適切な使用

  **4. 章間の整合性**
  - 前の章の最後と今回の章の導入が自然につながっているか
  - 次の章への橋渡しが適切か
  - 他の章で説明される概念への参照が正しいか

  **5. 技術的正確性**
  - OCI Generative AI Serviceの記述が正確か
  - マルチエージェントパターンの説明が正確か
  - コード例（ある場合）が動作するか

  **6. 図表の品質**
  - 各節に最低1つの図表があるか
  - 図表番号が正しいか（図X.Y形式）
  - キャプションが付いているか
  - 本文中で図表が参照されているか

  **7. 理解度チェック問題**
  - 3〜5問含まれているか
  - 章の内容を適切にカバーしているか

### ステップ3: 技術検証エージェントの起動

Task toolを使用してtech-verifierサブエージェントを起動してください:
- subagent_type: "tech-verifier"
- description: "Technical accuracy verification with web search"
- prompt: 以下の原稿に含まれる技術的な記述の正確性を、ウェブ検索を使って検証してください。

  対象原稿: `manuscript/ch[章番号]/ch[章番号].md`

  検証手順:
  1. 原稿を読み、技術的な主張・記述を抽出する（API仕様、サービス名、パラメータ、プロトコルの仕様、パターンの定義、バージョン情報等）
  2. 抽出した各主張について、web_searchツールで公式ドキュメントや信頼できるソースを検索する
  3. 原稿の記述とソースを照合し、不一致や古い情報を特定する

  特に以下を重点的に検証すること:
  - OCI Generative AI Serviceの仕様（利用可能モデル、API、リージョン、料金体系、Function Calling対応状況）
  - OCI各サービスの仕様（Resource Manager、OKE、Autonomous Database等）
  - MCP / A2Aプロトコルの仕様と現在のステータス
  - エージェントフレームワーク（LangChain, CrewAI, AutoGen等）の最新情報
  - Terraformプロバイダーの仕様

  出力形式（検証した主張ごと）:
  - **原稿の記述**: [原稿に書かれている内容]
  - **検索結果**: [web_searchで確認した内容とソースURL]
  - **判定**: 正確 / 要修正 / 要確認
  - **修正案**（要修正の場合）: [正しい記述]

このサブエージェントには `web_search` ツールの使用を許可する。

### ステップ4: 情報ソース検証エージェントの起動

Task toolを使用してcitation-verifierサブエージェントを起動してください:
- subagent_type: "citation-verifier"
- description: "Citation and source verification"
- prompt: 以下の原稿に含まれる「一般的な主張」に情報ソースが付いているかを確認し、付いている場合はその情報ソースの内容を検証してください。

  対象原稿: `manuscript/ch[章番号]/ch[章番号].md`

  検証手順:

  **ステップ1: ソースが必要な記述の検出**
  原稿を読み、以下のパターンに該当する記述を全て抽出する:
  - 「一般的には〜」「〜と言われている」「〜が主流である」
  - 「〜が広く採用されている」「〜がベストプラクティスとされている」
  - 「〜という調査によると」「〜の報告では」
  - 統計データ、ベンチマーク結果、市場シェアの記述
  - 特定の企業・プロジェクトの事例の記述

  **ステップ2: ソースの有無チェック**
  抽出した各記述に脚注（[^N]）または参考文献への参照があるかを確認する。
  ソースがない場合は「要ソース追加」として報告する。

  **ステップ3: ソースの内容検証**
  ソースが付いている場合、web_searchで以下を確認する:
  - そのソースが実在するか（URLが有効か、書籍が存在するか）
  - ソースの内容が原稿の主張を本当に裏付けているか
  - ソースが古すぎないか（5年以上前の情報は要確認）

  出力形式:
  ### [記述の概要]
  - **原稿の記述**: [該当箇所の引用]
  - **ソースの有無**: あり（[^N]）/ なし
  - **判定**: OK / 要ソース追加 / ソース不一致 / ソース不在
  - **対応案**: [具体的な修正案]

このサブエージェントには `web_search` ツールの使用を許可する。

### ステップ5: 理解度チェック問題フォーマット検証エージェントの起動

Task toolを使用してquiz-format-checkerサブエージェントを起動してください:
- subagent_type: "quiz-format-checker"
- description: "Quiz format compliance check"
- prompt: 以下の原稿の章末にある「理解度チェック」セクションが、共通フォーマットに準拠しているかを検証してください。

  対象原稿: `manuscript/ch[章番号]/ch[章番号].md`

  共通フォーマット要件:
  1. 「## 理解度チェック」見出しが存在する
  2. 各問題に以下の全要素が含まれている:
     - `### Q[N]. [問題タイトル]` 形式の見出し
     - `**種類**`: 「概念の確認」「判断問題」「設計問題」のいずれか
     - `**難易度**`: 「基礎」「応用」のいずれか
     - `**問題文**:` セクション
     - `**選択肢**` セクション（選択式の場合のみ。記述式は省略可）
     - `<details><summary>解答と解説</summary>` で囲まれた解答ブロック
     - 解答ブロック内に `**解答**:` `**解説**:` `**関連する節**:` の3要素
  3. 問題数が3〜5問の範囲内である
  4. 種類が偏りすぎていないか（3種のうち最低2種を含む推奨）

  出力形式:
  ### フォーマットチェック結果
  - **問題数**: [N]問（OK / 不足 / 過多）
  - **種類のバランス**: 概念確認×[N], 判断問題×[N], 設計問題×[N]（OK / 偏り）
  - **各問題の準拠状況**:
    - Q1: [OK / 不備の内容]
    - Q2: [OK / 不備の内容]
    - ...
  - **修正が必要な箇所**: [具体的な指摘]

### ステップ6: レビュー結果の統合と要約

ステップ2〜5の全エージェントの結果を統合し、ユーザーに報告します。

## 出力形式

```markdown
# 章レビュー結果

## 対象: 第[章番号]章 [章タイトル]

### 主な改善点

1. [改善点1] (優先度: 高/中/低)
2. [改善点2] (優先度: 高/中/低)
3. [改善点3] (優先度: 高/中/低)

### 各観点の評価

| 観点 | 評価 | 備考 |
|------|------|------|
| 仕様との整合性 | ○/△/× | [備考] |
| 用語の統一 | ○/△/× | [備考] |
| 文体・表記 | ○/△/× | [備考] |
| 章間の整合性 | ○/△/× | [備考] |
| 技術的正確性 | ○/△/× | [備考] |
| 図表の品質 | ○/△/× | [備考] |
| 理解度チェック | ○/△/× | [備考] |

### 総合評価

[1-5]/5

### 次のアクション

- [推奨対応1]
- [推奨対応2]
```

## 注意事項

- レビューは詳細な分析のため、数分かかる場合があります
- レビュー結果を`docs/feedbacks/`に保存したい場合は、`/fix-feedback`で修正ワークフローを実行できます
