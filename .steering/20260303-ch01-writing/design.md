# 執筆設計書

## 執筆アプローチ

第1章は書籍全体の出発点であり、読者が持つ「エージェント」の漠然としたイメージを構造化する役割を持つ。図表ファーストの原則に従い、各節で先に図を提示し、本文で解説するスタイルをとる。OCI固有の話題はまだ登場させず、概念的な理解に集中する。

## 節ごとの設計

### 1.1 エージェントの定義 ― チャットボットとの違い、自律性の意味

**導入**: 「エージェント」という用語が2024〜2025年に急速に普及した現状を示し、定義の必要性を提起する。

**図表設計**:
- 図1.1: チャットボットとエージェントの比較（Mermaid flowchart）
  - 左側：チャットボット（入力→LLM→応答の単純なフロー）
  - 右側：エージェント（目標→計画→ツール実行→観察→判断のループ）

**本文の構成**:
1. 「エージェント」の定義を提示
2. 図1.1でチャットボットとの違いを視覚化
3. 自律性（Autonomy）の意味を3つの特徴で整理
4. 「応答する」vs「目標を達成する」の対比

### 1.2 エージェントの構成要素 ― LLM / プロンプト / ツール / メモリ / プランニング

**導入**: エージェントを「ブラックボックス」ではなく分解可能な構造として捉える。

**図表設計**:
- 図1.2: エージェントの構成要素図（Mermaid flowchart）
  - 中心にLLM（推論エンジン）
  - 周囲にプロンプト、ツール、メモリ、プランニング
  - 外部世界との接点（ツール、メモリ）を強調

**本文の構成**:
1. 5つの構成要素の概観
2. 図1.2の解説（各要素の役割）
3. LLMは「推論エンジン」であってエージェントそのものではないことを強調
4. 各要素の相互関係（プランニング→ツール選択→メモリ更新）

### 1.3 ReActパターン ― 推論とアクションのループ

**導入**: エージェントの動作を1つのパターンとして抽象化する。

**図表設計**:
- 図1.3: ReActループのフロー図（Mermaid flowchart）
  - 思考（Reasoning）→ 行動（Acting）→ 観察（Observation）の循環
  - 停止条件の分岐を含む

**本文の構成**:
1. ReActの名称と由来
2. 図1.3でループ構造を解説
3. 具体例（「OCIのコンピュートインスタンスを一覧取得して」というタスクでの思考→行動→観察）
4. 停止条件の重要性

### 1.4 エージェントランタイムの役割 ― ループ制御、エラーハンドリング、停止条件

**導入**: ReActループを「誰が回しているか」という問いからランタイムの概念に入る。

**図表設計**:
- 図1.4: エージェントランタイムのアーキテクチャ図（Mermaid flowchart）
  - ランタイムがループ全体を包含
  - ループ制御、エラーハンドリング、停止条件管理の3責務

**本文の構成**:
1. ランタイムの定義と責務
2. 図1.4でアーキテクチャを解説
3. 3つの責務（ループ制御、エラーハンドリング、停止条件）の詳細
4. ランタイムは「安全装置」であるという位置づけ

### 1.5 エージェントフレームワークの全体像

**導入**: 構成要素とランタイムを理解した上で、主要フレームワークの位置づけを概観する。

**図表設計**:
- 表1.1: 主要エージェントフレームワーク比較表（Markdown table）
  - フレームワーク名、設計思想、特徴、得意領域

**本文の構成**:
1. フレームワークの役割（構成要素の組み合わせ方の提供）
2. 表1.1で主要フレームワークを比較
3. 本書の立場：特定フレームワークに依存しない
4. 概念理解がフレームワーク選択の基盤になる

## 章の導入

「エージェント（Agent）という言葉を聞かない日はなくなった」という現状認識から始め、この言葉が指す範囲の曖昧さを指摘する。本章の目的（定義の明確化と構成要素の分解）を提示する。

## 章の結び

エージェントの全体像を概観したことを確認し、構成要素のうち特に重要な「ツール」と「コンテキスト」を深掘りする必要性を示す。「エージェントの構成要素を概観した。次章ではその中核をなすツールとコンテキストに焦点を当てる」で締める。

## 理解度チェック問題の設計

1. エージェントとチャットボットの本質的な違いを3つ挙げよ（種類: 概念確認）
2. エージェントの5つの構成要素を列挙し、それぞれの役割を説明せよ（種類: 概念確認）
3. ReActパターンのループが停止する条件にはどのようなものがあるか（種類: 概念確認）
4. あるシステムがエージェントと呼べるかどうかを判断する基準を述べよ（種類: 判断問題）
