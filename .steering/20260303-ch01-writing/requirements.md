# 要求内容

## 概要

「エージェント」という曖昧な用語を正確に定義し、構成要素を分解する。チャットボットとの違いを明確にし、エージェントの全体像を把握する第1章を執筆する。

## 対象

- **章番号**: 第1章
- **対象節**: 1.1 〜 1.5
- **章仕様書**: `docs/chapter-specs/ch01-spec.md`

## 執筆内容

### 1. 1.1 エージェントの定義 ― チャットボットとの違い、自律性の意味
- エージェントの3つの特徴（自律性、ツール使用、計画能力）を定義
- チャットボットとの本質的な違いを「応答する」vs「目標を達成する」で整理
- 図1.1: チャットボットとエージェントの比較図

### 2. 1.2 エージェントの構成要素 ― LLM / プロンプト / ツール / メモリ / プランニング
- 5つの構成要素を分解して解説
- LLMは「推論エンジン」であり、エージェントそのものではないことを強調
- 図1.2: エージェントの構成要素図

### 3. 1.3 ReActパターン ― 推論とアクションのループ
- ReAct（Reasoning + Acting）のループ構造を解説
- 思考→行動→観察の循環
- 図1.3: ReActループのフロー図

### 4. 1.4 エージェントランタイムの役割
- ランタイムの責務（ループ制御、エラーハンドリング、停止条件）を整理
- ランタイムは「安全装置」であるという位置づけ
- 図1.4: エージェントランタイムのアーキテクチャ図

### 5. 1.5 エージェントフレームワークの全体像
- LangChain, CrewAI, AutoGen, LlamaIndex等の位置づけを概観
- 特定フレームワークに依存せず概念理解を重視
- 表1.1: 主要エージェントフレームワーク比較表
