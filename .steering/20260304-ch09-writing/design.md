# 第9章 設計方針

## 全体アプローチ
- 第I部・II部の理論を「OCI上の実装」に落とし込む
- コード例は疑似コードで構造理解を優先
- 各節で対応する理論章（第1,2,7章）を明示的に参照

## 節ごとの方針

### 9.1 アーキテクチャ設計
- エージェント構成要素→OCIサービスのマッピング表現
- ADR（アーキテクチャ決定記録）の考え方に言及

### 9.2 ReActループの実装
- 推論→アクション選択→実行→観察のフロー図
- 終了条件: 目標達成、最大ステップ数、タイムアウト
- Python疑似コードで基本構造を提示

### 9.3 ツールの実装
- カテゴリ別にツールパターンを整理（OCI操作、外部API、ファイル操作等）
- ツール定義の書き方（LLMが正しく選択するための説明文設計）

### 9.4 MCPサーバーとしての公開
- stdio / SSE / Streamable HTTPの選択基準
- ch10でのマルチエージェント化への布石

### 9.5 認証の設計
- Resource Principal / Instance Principal / APIキーの使い分け
- Dynamic Group + IAMポリシーの設計パターン

### 9.6 エラーハンドリングとリトライ
- エラー分類: 一時的 vs 永続的
- 指数バックオフ＋ジッターのリトライ戦略
- LLM固有エラーとOCI API固有エラーの分類

### 9.7 コンテキスト管理の実装
- コンテキストウィンドウの予算管理
- 古い会話履歴の要約・圧縮戦略
