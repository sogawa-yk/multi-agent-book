# Design: 第6章フィードバック修正

## 修正アプローチ

### 6.2節の再構成
現在の6.2節は「共有メモリ vs メッセージパッシング」を状態共有の二大アプローチとして説明しているが、これらが主に短期的な状態共有（タスク実行中の情報共有）に適する手法であることが不明確。

**修正方針:**
1. 6.2節の冒頭で、状態共有方法の全体像を示す
   - 共有メモリ（ブラックボード） → 主に短期共有コンテキストに適する
   - メッセージパッシング → 主に短期共有コンテキストに適する
   - 永続ストア（RDB、ベクトルDB、ファイルストア） → 主に長期記憶に適する
2. 各方法の説明の中で、短期/長期への適性を明示
3. 各方法に対応する実装コンポーネントを具体的に記述

### Mermaid色の変更
全Mermaid図のfill色をより濃い、コントラストの高い色に変更:
- `#e8f4fd` → `#1a73e8`（青系）+ 白文字
- `#fff3cd` → `#f9a825`（黄系）
- `#d4edda` → `#2e7d32`（緑系）+ 白文字
- `#f8d7da` → `#c62828`（赤系）+ 白文字

ただし、印刷書籍のグレー背景を考慮すると、明度差でコントラストを確保する方が適切。
濃い背景色+白文字、または明るい背景色+黒文字で統一。

### 用語の使い分け
- 6.2節: 短期/長期の区別を冒頭で明示し、以降の説明で適切に使い分ける
- 6.3節、6.4節はすでに短期/長期が分かれているため大きな変更不要
- まとめ節も同様に修正
