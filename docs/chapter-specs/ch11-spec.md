# 第11章 OCI固有のサービスを活用するエージェント設計 ― 仕様書

## 章の概要

**章の目的**: OCI特有のサービスをエージェントのツール/基盤として活かす設計を学ぶ。各サービスの特性とエージェント連携のパターンを理解する。

**目標ページ数**: 20p

**所属する部**: 第III部 OCI上での実践

## 前提知識

この章を読むために必要な前提:
- 第10章の内容（マルチエージェントシステムの構築）
- OCIの基本的なサービス知識

## 節の構成

### 11.1 Resource Manager ― Terraform実行のマネージドサービスとしての活用

**概要**: OCI Resource Managerを使ったTerraform実行の自動化を解説する。エージェントがスタックの作成・計画・適用を制御するパターンを示す。

**図表**:
- 図11.1: Resource Managerとエージェントの連携アーキテクチャ図

**キーポイント**:
- Resource Managerはエージェントにとって「インフラ操作ツール」
- 計画（plan）→確認→適用（apply）のフローでHuman-in-the-Loopを組み込む

### 11.2 OKE ― Kubernetesクラスタ操作のツール化

**概要**: OKE（Oracle Container Engine for Kubernetes）のクラスタ操作をエージェントのツールとしてラップする設計を解説する。

**図表**:
- 図11.2: OKEクラスタ操作のツール化アーキテクチャ図

**キーポイント**:
- Kubernetes APIをエージェントから操作するパターン
- クラスタの作成・スケーリング・デプロイメント管理

### 11.3 Autonomous Database ― エージェントの知識ベース（Vector Search, JSON, SQL）

**概要**: Autonomous Databaseをエージェントの知識ベースとして活用する設計を解説する。Vector Search（セマンティック検索）、JSONドキュメント、SQL問い合わせの3つの活用パターンを示す。

**図表**:
- 図11.3: Autonomous Databaseの3つの活用パターン図

**キーポイント**:
- 1つのサービスで構造化データ・非構造化データ・セマンティック検索をカバー
- エージェントの長期記憶として強力な選択肢

### 11.4 Streaming / Queue ― エージェント間の非同期通信基盤

**概要**: OCI StreamingとQueueをエージェント間の非同期通信基盤として活用する設計を解説する。

### 11.5 Logging / Monitoring ― エージェントの行動ログの収集

**概要**: OCI API Gatewayを使ってエージェントを外部に公開し、アクセス制御を行う設計を解説する。

- 図11.7: API Gatewayによるエージェント公開のアーキテクチャ図

## 章末の理解度チェック問題

### 問題の方針
- 問題数: 4問
- 問題の種類: 概念の確認、判断問題、設計問題

### 問題案
1. OCIサービスとエージェント機能のマッピングを5つ挙げよ（種類: 概念の確認）
2. StreamingとQueueの使い分けの判断基準を述べよ（種類: 判断問題）
3. エージェントの知識ベースとしてAutonomous Databaseを選択する利点を述べよ（種類: 概念の確認）
4. 3つのエージェントから成るシステムにおいて、各エージェントのVaultアクセス権限を設計せよ（種類: 設計問題）
