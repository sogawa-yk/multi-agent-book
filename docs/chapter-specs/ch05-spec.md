# 第5章 エージェント間の通信 ― MCP, A2A, そしてその先 ― 仕様書

## 章の概要

**章の目的**: エージェント間通信のプロトコルと設計選択を理解する。MCPの拡張的活用とA2Aプロトコルを中心に、同期/非同期通信の設計判断を学ぶ。

**目標ページ数**: 20p

**所属する部**: 第II部 マルチエージェントの理論

## 前提知識

この章を読むために必要な前提:
- 第4章の内容（協調パターンの種類と特徴）
- 第2章の内容（MCP、Function Callingの基礎）

**概要**: マルチエージェントシステムにおける通信の3つの設計課題（内容、形式、方法）を整理する。シングルエージェントにはなかった課題を明確にする。

### 5.2 MCPの拡張的活用 ― ツール提供からエージェント連携へ

**概要**: 第2章で学んだMCPを、エージェント間連携に拡張的に活用する方法を解説する。あるエージェントが別のエージェントをMCPサーバーとして利用するパターンを示す。

- ただし、MCPはリクエスト-レスポンス型であり、対等な通信には限界がある

### 5.5 非同期通信 ― キューイング、イベント駆動

**概要**: キューやイベントを使った非同期通信パターンを解説する。処理時間が長いタスクや、エージェント間のペースの違いに対応する方法を示す。

**概要**: 同期通信と非同期通信の使い分けの判断基準を解説する。レイテンシ、信頼性、結合度の観点からの選択指針を示す。

## 章末の理解度チェック問題

### 問題の方針
- 問題数: 4問
- 問題の種類: 概念の確認、判断問題

### 問題案
1. MCPとA2Aの役割の違いを「ツール提供」と「タスク委任」の観点で説明せよ（種類: 概念の確認）
2. A2AのAgent Cardはどのような情報を含み、なぜ必要か（種類: 概念の確認）
3. 次のケースで同期通信と非同期通信のどちらを選ぶべきか、理由と共に述べよ（種類: 判断問題）
4. サーキットブレーカーパターンがマルチエージェントシステムで有効な理由を説明せよ（種類: 概念の確認）
