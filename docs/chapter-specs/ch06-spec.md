# 第6章 状態管理とメモリアーキテクチャ ― 仕様書

## 章の概要

**章の目的**: マルチエージェントにおける状態共有と永続化の設計を学ぶ。複数のエージェントがどのように情報を共有し、一貫した処理を実現するかを理解する。

**目標ページ数**: 18p

**所属する部**: 第II部 マルチエージェントの理論

## 前提知識

この章を読むために必要な前提:
- 第4章の内容（協調パターン）
- 第2章の内容（コンテキスト管理、短期記憶と長期記憶）

## 節の構成

### 6.1 マルチエージェントにおける状態の種類 ― タスク状態、共有コンテキスト、エージェント固有状態

**概要**: マルチエージェントシステムにおける状態を3種類に分類する。タスク状態（進捗管理）、共有コンテキスト（エージェント間で共有する情報）、エージェント固有状態（各エージェントの内部状態）を整理する。

**図表**:
- 図6.1: 状態の3分類と各エージェントとの関係図

**キーポイント**:
- 全ての状態を共有する必要はない
- 状態の分類が設計の出発点

### 6.2 共有メモリ vs メッセージパッシング ― 状態共有の2つのアプローチ

**概要**: 状態を共有する2つの基本アプローチ（共有メモリとメッセージパッシング）を比較する。それぞれの利点・欠点と適用場面を解説する。

**図表**:
- 図6.2: 共有メモリとメッセージパッシングの比較図

**キーポイント**:
- 共有メモリ：アクセスが容易だが、競合と整合性の管理が必要
- メッセージパッシング：分離度が高いが、通信コストが増加

### 6.3 短期共有コンテキスト ― ワーキングメモリ、ブラックボード

**概要**: エージェント間で短期的に共有するコンテキストの実装パターンを解説する。ワーキングメモリ（一時的な共有空間）とブラックボードパターンを紹介する。

**図表**:
- 図6.3: ブラックボードパターンのアーキテクチャ図

**キーポイント**:
- ブラックボードは「中央の掲示板」として機能する
- エージェント間の間接的な協調を実現する

### 6.4 長期記憶 ― ベクトルDB、ナレッジグラフ、構造化ストレージ

**概要**: エージェントシステム全体で永続化する長期記憶の実装選択肢を解説する。ベクトルDB（セマンティック検索）、ナレッジグラフ（関係性の表現）、構造化ストレージ（RDB/NoSQL）のそれぞれの特徴を示す。

**図表**:
- 表6.1: 長期記憶の実装選択肢比較表

**キーポイント**:
- 用途によって最適なストレージが異なる
- 複数のストレージを組み合わせることも一般的

### 6.5 状態の永続化戦略 ― チェックポイント、イベントソーシング

**概要**: エージェントの処理状態を永続化する戦略を解説する。チェックポイント（スナップショット保存）とイベントソーシング（イベント列の記録）のそれぞれの手法を示す。

**図表**:
- 図6.4: チェックポイントとイベントソーシングの比較図

**キーポイント**:
- チェックポイント：復旧が容易だが、中間状態が失われる
- イベントソーシング：完全な履歴が残るが、再生コストが高い

### 6.6 コンテキストの分離と漏洩防止 ― エージェントが見てよい情報の制御

**概要**: エージェントが見るべき情報と見るべきでない情報を制御する設計を解説する。セキュリティとプライバシーの観点からのアクセス制御を扱う。

**図表**:
- 図6.5: コンテキスト分離のアーキテクチャ図

**キーポイント**:
- 「何でも見える」は設計ミス
- 最小権限の原則はコンテキストにも適用する

### 6.7 OCI上の選択肢 ― Object Storage / Autonomous DB / Cache with Redis

**概要**: OCI上で利用可能な状態管理サービスを整理する。Object Storage（大容量データ）、Autonomous Database（構造化データ + Vector Search）、Cache with Redis（高速アクセス）の使い分けを示す。

**図表**:
- 表6.2: OCI上の状態管理オプション比較表

**キーポイント**:
- OCI上のサービスと状態管理パターンのマッピング
- 第III部での実装の準備

## 章末の理解度チェック問題

### 問題の方針
- 問題数: 4問
- 問題の種類: 概念の確認、判断問題、設計問題

### 問題案
1. マルチエージェントにおける状態の3種類を説明せよ（種類: 概念の確認）
2. 共有メモリとメッセージパッシングの利点・欠点を比較せよ（種類: 概念の確認）
3. 次のユースケースに対して、チェックポイントとイベントソーシングのどちらが適切か判断し理由を述べよ（種類: 判断問題）
4. 3つのエージェントが協調するシステムにおいて、各エージェントがアクセスすべき状態とアクセスすべきでない状態を設計せよ（種類: 設計問題）
