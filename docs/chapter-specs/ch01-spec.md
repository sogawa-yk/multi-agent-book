# 第1章 AIエージェントとは何か ― 改めて定義する ― 仕様書

## 章の概要

**章の目的**: 「エージェント」という曖昧な用語を正確に定義し、構成要素を分解する。チャットボットとの違いを明確にし、エージェントの全体像を把握する。

**目標ページ数**: 14p

**所属する部**: 第I部 シングルエージェントの深い理解

## 前提知識

この章を読むために必要な前提:
- LLMの基本的な概念（プロンプト、トークン、推論）
- REST APIの基本（HTTPメソッド、JSON）

## 節の構成

### 1.1 エージェントの定義 ― チャットボットとの違い、自律性の意味

**概要**: エージェントの定義を明確にする。チャットボットとの本質的な違い（自律性、ツール使用、計画能力）を整理し、「エージェント」が指す範囲を読者と共有する。

**図表**:
- 図1.1: チャットボットとエージェントの比較図（入力→処理→出力のフローの違い）

**キーポイント**:
- エージェントの3つの特徴：自律性、ツール使用、計画能力
- チャットボットは「応答する」、エージェントは「目標を達成する」

### 1.2 エージェントの構成要素 ― LLM / プロンプト / ツール / メモリ / プランニング

**概要**: エージェントを構成する5つの要素を分解して解説する。各要素の役割と、要素間の関係を明確にする。

**図表**:
- 図1.2: エージェントの構成要素図（5要素の関係を示すブロック図）

**キーポイント**:
- LLMは「推論エンジン」であり、エージェントそのものではない
- 5要素のうち、ツールとメモリが外部世界との接点

### 1.3 ReActパターン ― 推論とアクションのループ

**概要**: エージェントの基本動作パターンであるReAct（Reasoning + Acting）を解説する。思考→行動→観察のループがどのように問題解決を進めるかを示す。

**図表**:
- 図1.3: ReActループのフロー図（思考→行動→観察の循環）

**キーポイント**:
- ReActは「考えてから動く」パターン
- ループの停止条件が重要（無限ループの回避）

### 1.4 エージェントランタイムの役割 ― ループ制御、エラーハンドリング、停止条件

**概要**: エージェントを実行するランタイムの責務を整理する。ループ制御、エラーハンドリング、停止条件管理など、「エージェントを安全に動かす」ための仕組みを解説する。

**図表**:
- 図1.4: エージェントランタイムのアーキテクチャ図

**キーポイント**:
- ランタイムはエージェントの「安全装置」
- 停止条件の設計がエージェントの信頼性を決める

### 1.5 エージェントフレームワークの全体像 ― LangChain, CrewAI, AutoGen, LlamaIndex等の位置づけ

**概要**: 主要なエージェントフレームワークを概観し、それぞれの設計思想と位置づけを整理する。本書では特定のフレームワークに依存しないが、概念の理解を助けるために参照する。

**図表**:
- 表1.1: 主要エージェントフレームワーク比較表

**キーポイント**:
- フレームワークは構成要素の組み合わせ方が異なる
- 本書では概念を理解し、OCI上での実装に活かす

## 章末の理解度チェック問題

### 問題の方針
- 問題数: 4問
- 問題の種類: 概念の確認、判断問題

### 問題案
1. エージェントとチャットボットの本質的な違いを3つ挙げよ（種類: 概念の確認）
2. エージェントの5つの構成要素を列挙し、それぞれの役割を説明せよ（種類: 概念の確認）
3. ReActパターンのループが停止する条件にはどのようなものがあるか（種類: 概念の確認）
4. あるシステムがエージェントと呼べるかどうかを判断する基準を述べよ（種類: 判断問題）
