# 第1章 AIエージェントとは何か ― 改めて定義する ― 仕様書

## 章の概要

**章の目的**: AIエージェントの定義を明確にし、構成要素・動作パターン・ランタイム・フレームワークの全体像を体系的に整理することで、以降の章の土台を築く。
**目標ページ数**: 14p
**所属する部**: 第I部 シングルエージェントの深い理解

## 前提知識

- LLM（大規模言語モデル）の基本概念と利用経験（チャット、API呼び出し程度）
- エージェントの基本概念（LLMにツールを与えて自律的にタスクを実行させる仕組み）の大まかな理解
- REST APIの基本概念

## 節の構成

### 1.1 エージェントの定義

**概要**: 「チャットボット」「アシスタント」「エージェント」の違いを明確にし、本書におけるエージェントの定義を確立する。LLMを推論エンジンとし、ツール呼び出しとプランニング能力を持つ自律的なソフトウェアエンティティとして定義する。自律性のレベル（Agentic Spectrum）を示し、読者が既存のシステムをスペクトラム上に位置づけられるようにする。

**図表**:
- 図1.1: 自律性のスペクトラム（チャットボット → アシスタント → エージェントの連続体）

**キーポイント**:
- エージェントの本質は「目標に向けた自律的な行動の連鎖」にある
- チャットボットとの決定的な違いは「ツール呼び出し」と「ループによる自律的な行動制御」
- 自律性はスペクトラムであり、二者択一ではない

### 1.2 構成要素

**概要**: エージェントを構成する4つの要素（LLM・ツール・プランニング・メモリ）を分解して説明する。各要素が果たす役割と、要素間の関係を明確にする。この分解は第2章（ツールとコンテキスト）および第3章（限界分析）の基盤となる。

**図表**:
- 図1.2: エージェントの構成要素図（LLM・ツール・プランニング・メモリの関係）

**キーポイント**:
- LLMは「推論エンジン」であり、エージェントそのものではない
- ツールはエージェントの「手」であり、外部世界との接点
- プランニングはタスク分解と実行順序の決定を担う
- メモリは過去の情報を保持し、文脈を維持する

### 1.3 ReActパターン

**概要**: エージェントの基本動作パターンであるReAct（Reasoning + Action）を解説する。思考→行動→観察のループがどのように機能するかを示し、このパターンが以降のすべてのエージェント設計の基盤であることを明確にする。

**図表**:
- 図1.3: ReActループのフロー図（思考→行動→観察のサイクル）

**キーポイント**:
- ReActは「推論」と「行動」を交互に繰り返す基本パターン
- 各ステップで「思考（Thought）」「行動（Action）」「観察（Observation）」が明示される
- ループの終了条件（目標達成 or 最大ステップ到達）の重要性
- このパターンがシングルでもマルチでもエージェントの基本単位となる

### 1.4 エージェントランタイムの役割

**概要**: エージェントの実行ループを制御するランタイムの役割を説明する。ランタイムが担う責務（ループ制御、ツール呼び出しの仲介、エラーハンドリング、停止条件管理）を整理し、「エージェント=LLM+ツール+ランタイム」という構造を理解させる。

**図表**:
- なし（1.2の構成要素図と1.3のReActフロー図で補完）

**キーポイント**:
- ランタイムはReActループの「実行環境」を提供する
- ツール呼び出しの仲介、結果のLLMへの返却を担う
- エラーハンドリングと停止条件の管理がランタイムの重要な責務
- ランタイムの設計がエージェントの信頼性を左右する

### 1.5 エージェントフレームワークの全体像

**概要**: 主要なエージェントフレームワーク（LangGraph、CrewAI、AutoGen、OpenAI Agents SDK等）を俯瞰し、各フレームワークの位置づけと特徴を整理する。本書は特定フレームワークに依存しないが、概念の具体的なイメージを持たせるための参照点として活用する。

**図表**:
- 表1.1: 主要エージェントフレームワーク比較表

**キーポイント**:
- フレームワークは構成要素（LLM・ツール・ランタイム）の実装を提供する
- フレームワークごとにマルチエージェント対応のレベルが異なる
- 本書は特定フレームワークに依存せず、共通の概念を学ぶ
- 読者はClaude Codeを使って実装するため、フレームワークの選択は柔軟

## 章末の理解度チェック問題

1. エージェントとチャットボットの決定的な違いを2つ挙げ、それぞれ説明せよ。
2. エージェントの4つの構成要素を挙げ、それぞれの役割を簡潔に述べよ。
3. ReActパターンにおける「思考」「行動」「観察」の各ステップの役割を説明せよ。
4. エージェントランタイムが担う責務を3つ挙げよ。
5. エージェントフレームワークが提供する機能と、フレームワークに依存しない設計の利点を述べよ。
