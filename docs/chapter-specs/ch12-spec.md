# 第12章 ケーススタディ ― OKEクラスタの自動構築システム ― 仕様書

## 章の概要

**章の目的**: 第I〜III部の知識を統合し、現実的なユースケースの設計判断を追体験する。OKEクラスタの自動構築を題材に、マルチエージェントシステムの設計から実装までを通して示す。

**目標ページ数**: 24p

**所属する部**: 第III部 OCI上での実践

## 前提知識

この章を読むために必要な前提:
- 第7章の内容（設計原則）
- 第10章の内容（マルチエージェントシステムの構築）
- 第11章の内容（OCI固有サービスの活用）

## 章末の理解度チェック問題

### 問題の方針
- 問題数: 4問
- 問題の種類: 判断問題、設計問題

### 問題案
1. 本ケーススタディでオーケストレーター型を選んだ理由と、スーパーバイザー型を選んだ場合の違いを述べよ（種類: 判断問題）
2. IaC生成でテンプレートベースとLLM生成を組み合わせる利点を述べよ（種類: 判断問題）
3. デプロイ中にTerraform applyが失敗した場合のリカバリフローを設計せよ（種類: 設計問題）
4. 本ケーススタディのエージェント粒度を変更する場合（3エージェントに統合する、8エージェントに分割する）のトレードオフを述べよ（種類: 設計問題）
