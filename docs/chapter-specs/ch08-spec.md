# 第8章 OCI Generative AI Service入門 ― 仕様書

## 章の概要

**章の目的**: OCI Generative AI Serviceの全体像と利用方法を理解し、以降の実践章（第9〜12章）でエージェントを構築するための技術的基盤を確立する。
**目標ページ数**: 20p
**所属する部**: 第III部 OCI上での実践

## 前提知識

本章を読む前に、読者が理解している必要がある概念:

- **第2章**: Function Callingの仕組み、コンテキストウィンドウの概念、MCPの基本
- **第7章（第II部全体）**: マルチエージェントの理論を一通り理解していること。本章からは実践に入る
- **OCI基礎**: OCIのコンソール操作、コンパートメント、IAMの基本（読者の前提知識として想定）

### 8.5 Dedicated AI Cluster vs On-Demand

**概要**: OCI GenAI Serviceの二つの利用形態 ― Dedicated AI ClusterとOn-Demand ― の違いと選択基準を整理する。コスト、性能、可用性の観点から比較する。

**図表**:
- 表8.2: Dedicated AI Cluster vs On-Demand 比較表

**キーポイント**:
- Dedicated AI Cluster: 専用のGPUリソースを確保。安定した性能、カスタムモデルのホスティングが可能
- On-Demand: 共有リソースを従量課金で利用。小規模・検証用途に適する
- コスト比較: 利用パターンに応じた損益分岐点
- エージェントシステムの本番運用ではDedicated AI Clusterが推奨される場面

## 章末の理解度チェック問題

1. OCI Generative AI Serviceが提供する主要機能を列挙し、エージェント構築においてそれぞれがどのように活用されるかを説明せよ。
2. エージェント用途のモデル選択において、コンテキスト長とFunction Calling対応が重要な理由を説明せよ。
3. Dedicated AI ClusterとOn-Demandの使い分けを、エージェントシステムの開発フェーズ（検証/本番）に応じて説明せよ。
4. Function Callingに対応していないモデルをエージェントに利用する場合、どのような回避策が考えられるか述べよ。
