# 第10章 マルチエージェントシステムの構築 ― 仕様書

## 章の概要

**章の目的**: 複数エージェントを協調させるシステムをOCI上に構築する。本書の実践的核心となる章。

**目標ページ数**: 28p

**所属する部**: 第III部 OCI上での実践

## 前提知識

この章を読むために必要な前提:
- 第4章の内容（協調パターン）
- 第5章の内容（エージェント間通信）
- 第6章の内容（状態管理）
- 第9章の内容（シングルエージェントの構築）

## 節の構成

**概要**: システムを構成する各エージェントの責務を定義し、エージェント間のインターフェース（入出力）を設計する。第7章の単一責任原則と粒度設計を適用する。

## 章末の理解度チェック問題

### 問題の方針
- 問題数: 5問
- 問題の種類: 概念の確認、判断問題、設計問題

### 問題案
1. オーケストレーターがタスクを分解する際に考慮すべき要素を3つ挙げよ（種類: 概念の確認）
2. 直接呼び出しとメッセージキューの利点・欠点を比較せよ（種類: 概念の確認）
3. 次のエージェント構成に対して、Container Instances / OKE / Functionsのどれを選ぶべきか判断せよ（種類: 判断問題）
4. Human-in-the-Loopの承認リクエストがタイムアウトした場合の適切な処理を設計せよ（種類: 設計問題）
5. 3つのエージェントが共有状態を利用するシステムにおいて、競合が発生した場合の対処戦略を述べよ（種類: 設計問題）
