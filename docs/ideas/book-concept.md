# AIエージェント開発入門 ― シングルからマルチへ
## OCI Generative AI Serviceで学ぶマルチエージェントの設計と実装

**形態**: 技術同人誌（技術書典向け、B5判）  
**想定ページ数**: 約250〜300ページ  
**対象読者**: OCI技術者。エージェントの基本（LLM + ツール呼び出し）は理解済み。マルチエージェントを体系的に学びたい。  
**文体**: 常体（である調）  
**LLM環境**: OCI Generative AI Service（Cohere Command R+, Meta Llama等）  
**コード言語**: Python（OCI SDK for Python）  

---

## 書籍の到達目標

読了後、以下ができるようになること：

1. シングルエージェントの「どこに限界があるか」を構造的に説明できる
2. マルチエージェントの主要パターンを理解し、ユースケースに応じて選択できる
3. エージェント間の通信プロトコル（MCP, A2A）の役割と違いを整理できている
4. OCI Generative AI Service + OCIサービス群を使って、マルチエージェントシステムを設計・構築できる
5. テスト・デバッグ・運用の勘所を押さえている

---

## 全体構成

本書は4部構成をとる。

- **第I部（第1〜3章）**: 基礎の再確認と限界の認識。シングルエージェントを深く理解し、「なぜマルチが必要か」を腹落ちさせる。
- **第II部（第4〜7章）**: マルチエージェントの理論。パターン、通信、状態管理、設計原則を体系的に学ぶ。
- **第III部（第8〜12章）**: OCI上での実践。OCI Generative AI Serviceを中心に、実際にマルチエージェントシステムを設計・構築する。
- **第IV部（第13〜15章）**: 運用と発展。テスト、可観測性、そして今後の展望。

## 第II部　マルチエージェントの理論

### 第4章　マルチエージェントの協調パターン（約24ページ）

**章の目的**: 協調パターンを体系的に理解し、ユースケースに応じた選択ができるようになる

- 4.1 パターン分類の軸 ― 制御の集中度 × 通信の方向性
- 4.2 直列パイプライン（Sequential）
  - 構造と特徴
  - 適するユースケース：段階的な変換、承認フロー
  - OCI例：ヒアリング → 設計 → Terraform生成 → Resource Managerデプロイ
- 4.3 並列ファンアウト/ファンイン（Parallel）
  - 構造と特徴
  - 適するユースケース：独立した調査、同時検証
  - OCI例：複数リージョンの同時ヘルスチェック
- 4.4 オーケストレーター型（Orchestrator）
  - 構造と特徴
  - 動的タスク分配、サブタスクの生成と統合
  - OCI例：インフラ構築の全体制御
- 4.5 スーパーバイザー型（Supervisor / Hierarchical）
  - オーケストレーターとの違い ― 監督と委任
  - 階層構造、エスカレーション
- 4.6 コレオグラフィ型（Choreography / Peer-to-Peer）
  - 中央制御なしの自律協調
  - A2Aプロトコルとの関係
- 4.7 評価者ループ（Evaluator / Critic）
  - 生成→評価→修正のサイクル
  - 品質保証パターンとしての活用
- 4.8 パターンの組み合わせ ― 実システムはハイブリッド
- 4.9 パターン選定ガイド ― ディシジョンツリーとトレードオフ表

**図表**: 各パターンのアーキテクチャ図（6パターン）、パターン比較表、ディシジョンツリー

## 執筆上の方針

- **図表ファースト**: 各節の核となるメッセージは図で表現し、本文は図の解説として書く
- **OCI環境前提**: コード例・構成例はすべてOCI Generative AI Service + OCI SDKで統一
- **コードは最小限**: 設計概念の理解に必要な疑似コードや設定抜粋に留める。完全実装は載せない
- **既存フレームワークへの言及**: LangChain, CrewAI等は概念の理解を助けるために参照するが、特定フレームワーク依存にはしない
- **各章末に「理解度チェック」**: 3〜5問の確認問題で、読んだ内容の定着を促す
- **Galley開発の知見**: 直接的な言及は避けるが、実体験に基づくリアルな設計判断をケーススタディに反映する
