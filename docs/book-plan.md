# 書籍企画書 (Book Plan)

## 書籍概要

### タイトル
**AIエージェント開発入門** ― シングルからマルチへ
OCI Generative AI Serviceで学ぶマルチエージェントの設計と実装

### 形態
- 判型: B5
- 想定ページ数: 約300ページ
- 配布形態: 技術同人誌（技術書典向け）

### コンセプト
- **基礎から実践へ**: シングルエージェントの深い理解からマルチエージェントの設計・構築まで、段階的に学べる構成
- **OCI実践**: OCI Generative AI Serviceを中心に、OCIサービス群を活用した具体的なマルチエージェントシステムの構築手法を提供
- **図表ファースト**: 各節の核となるメッセージを図で表現し、本文は図の解説として書く。視覚的に理解できる技術書

### ビジョン
本書は、AIエージェントの基本を理解した読者が「マルチエージェント」という次のステップに進むための実践的なガイドである。単一エージェントの限界を構造的に理解した上で、マルチエージェントの協調パターン・通信プロトコル・状態管理を体系的に学び、OCI Generative AI Serviceを用いて実際にシステムを構築できるようになることを目指す。理論と実践を往復しながら、読者が自信を持ってマルチエージェントシステムを設計・構築できる知識と判断力を身につけられる一冊とする。

## 対象読者

### プライマリーペルソナ
- OCIを業務で利用するクラウドエンジニア・ソリューションアーキテクト
- LLMの基本的な利用経験あり（チャット、API呼び出し程度）
- エージェントの基本概念（LLM + ツール呼び出し）は理解済み
- マルチエージェントの設計パターンや実装手法を体系的に学びたい

### 前提知識
読者が本書を読む前に理解していることを前提とする:
- Pythonの基本的なプログラミング能力
- LLM（大規模言語モデル）の基本概念と利用経験
- OCIの基本的なサービス（コンピュート、ネットワーク、IAM等）の理解
- REST APIの基本概念
- エージェントの基本概念（LLMにツールを与えて自律的にタスクを実行させる仕組み）

### 前提としない知識
本書で説明する:
- マルチエージェントの協調パターン（直列、並列、オーケストレーター等）
- エージェント間通信プロトコル（MCP, A2A）の詳細
- OCI Generative AI Serviceの具体的な利用方法
- マルチエージェントシステムの状態管理・テスト・運用手法

## 到達目標

読了後、読者が以下をできるようになること:

1. シングルエージェントの「どこに限界があるか」を構造的に説明できる
2. マルチエージェントの主要パターン（直列、並列、オーケストレーター、スーパーバイザー、コレオグラフィ、評価者ループ）を理解し、ユースケースに応じて選択できる
3. エージェント間の通信プロトコル（MCP, A2A）の役割と違いを整理できている
4. OCI Generative AI Service + OCIサービス群を使って、マルチエージェントシステムを設計・構築できる
5. マルチエージェントシステムのテスト・デバッグ・運用の勘所を押さえている

## LLM環境

- **使用サービス**: OCI Generative AI Service
- **利用可能モデル**: Cohere Command R+、Meta Llama等（OCI上で提供されるモデル）
- **コード例の言語**: Python（OCI SDK for Python）※構造理解に必要な場合のみ掲載
- **図表記法**: Mermaid（ダイアグラム系）
- **実装ツール**: 読者はClaude Codeを使って実装を行う前提

## 執筆方針

- **図表ファースト**: 各節の核となるメッセージは図で表現し、本文は図の解説として書く。ダイアグラム系の図はすべてMermaid記法で記述する
- **OCI環境前提**: 構成例・アーキテクチャ例はすべてOCI Generative AI Service + OCIサービス群を前提とする
- **コード例は原則不要**: 読者はClaude Codeを使って実装を行うため、コード例は基本的に載せない。ただし、構造やアーキテクチャの理解にコードがあった方が明らかにわかりやすい場合に限り、最小限のコード（疑似コード含む）を掲載する
- **既存フレームワークへの言及**: LangChain, CrewAI, AutoGen等は概念の理解を助けるために参照するが、特定フレームワーク依存にはしない
- **各章末に理解度チェック**: 3〜5問の確認問題で、読んだ内容の定着を促す
- **実体験ベースの設計判断**: Galley開発の知見を直接的には言及せず、リアルな設計判断としてケーススタディに反映する

## スコープ外

明示的にスコープ外とする項目:
- 特定のエージェントフレームワーク（LangChain, CrewAI等）の詳細なチュートリアル
- LLMのファインチューニングやモデル学習
- OCI以外のクラウドプラットフォーム（AWS, Azure, GCP）上での実装
- フロントエンド/UIの実装（エージェントのバックエンド設計に集中）
- 完全な本番システムのソースコード一式（読者はClaude Codeで実装するため、コード例は構造理解に必要な場合のみ）
- エージェントの倫理・法規制（簡潔な言及はするが、深い議論は行わない）

## 成功指標

- 読者がシングルエージェントとマルチエージェントの違い・使い分けを自信を持って説明できる
- 読者がOCI上でマルチエージェントシステムのプロトタイプを構築できるだけの知識を得られる
- 各章の理解度チェック問題に概ね正答できる水準の理解が得られる
- 技術書典で手に取った読者が「OCIでのエージェント開発の参考になった」と感じる
