# 改善してほしい点
- 状態＝短期記憶＋長期記憶であるが、状態の共有方法として述べられているブラックボードパターンとメッセージパッシングパターンが短期記憶で使う方法である
- 短期記憶と長期記憶のどちらにも共通する項の場合はそのように述べてもいいが、それぞれ固有の問題は状態という言葉を使って述べない。誤解を生む
- マーメイド表記がグレー背景でも見やすい色にする
- 状態に関して、概念的にこのような共有方法があり、この方法は長期記憶に適して、この方法は短期記憶に適するのような形で整理する
- 整理したそれぞれの状態共有方法に対して、実際のコンポーネントを説明する（RDBやkey-valueストア、ファイルなど）
